name: Commit style preference
description: User wants frequent commits with conventional commit format and an explanatory body
type: feedback
---

Commit early and often. Each logical unit of work (new file, passing test suite, bug fix) warrants its own commit.

**Why:** User explicitly requested this as a working norm for the project.

**How to apply:** Use conventional commit format (`feat:`, `fix:`, `test:`, `chore:`, etc.) with a subject line and a body that explains what changed and why. Do not batch unrelated changes into one commit.
